# 뉴스 Record 템플릿

## 파일명 규칙
`YYYY-MM-DD-{slug}.md` (예: `2026-06-23-claude-computer-use.md`)

## 구조

```markdown
# {뉴스 제목}

## 메타데이터
- **원문 URL**: {url}
- **소스**: {출처}
- **발행일**: YYYY-MM-DD
- **수집일**: YYYY-MM-DD
- **수집자**: {대구루|레노버}
- **카테고리**: {framework|model|tool|industry|research}
- **태그**: [태그1, 태그2]
- **중요도**: ⭐⭐⭐⭐⭐ (1~5)
- **신선도**: fresh | updated | stale (수집 시 fresh, 60일 후 stale)

## 핵심 요약
> {1-2문장 요약}

## 번역 (한국어)
{핵심 내용 한국어 번역, 3~5문단}

## 왜 중요한가?
{이 뉴스가 왜 중요한지 2~3문장으로 설명. 비기술자도 이해할 수 있게.}

## 심층 분석

### 기술 의미
{이 뉴스의 기술적 의미, 최소 3문장}

### 업계 영향
{AI 에이전트 생태계에 미치는 영향, 최소 3문장}

### 관련 프로젝트
- {관련 링크/프로젝트}

### 관련 뉴스
- [제목](../records/{파일명}.md) — {한 줄 설명}

## 원문 발췌
> {중요 문장 영어 원문, 2~3문장 이내}
```

## 품질 체크리스트 (Lint 기준)
- [ ] 메타데이터 모든 필드 채움 (중요도, 신선도 포함)
- [ ] 핵심 요약 1~2문장 (블록인용문)
- [ ] 번역 3문단 이상 (직역 금지, 자연스러운 한국어)
- [ ] 심층 분석 3개 섹션 (기술 의미, 업계 영향, 관련 프로젝트)
- [ ] 관련 뉴스 교차 링크 (동일 카테고리 또는 관련 카테고리)
- [ ] 원문 발췌 2~3문장 (저작권 고려)
- [ ] 태그 3개 이상
- [ ] 카테고리 정확 (framework|model|tool|industry|research)
